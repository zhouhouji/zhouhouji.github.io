解决 Ubuntu 出现kernel panic – kernel panic not syncing vfs unable to mount root fs on unknown-block 0 0 – error

发表于 2021-01-04 分类于 bug
Ubuntu系统出现kernel panic – kernel panic not syncing vfs unable to mount root fs on unknown-block 0 0 – error 错误的解决办法

项目场景：

Ubuntu系统出现kernel panic – kernel panic not syncing vfs unable to mount root fs on unknown-block 0 0 – error 错误的解决办法

问题描述：

这个问题是突然出现的，大概情况是在服务器的其他程序正常远程运行的时候突然连接中断（大概率是直接宕机了），然后过去查看出现了下面的问题

原因分析：

在许多情况下，这是由于/ boot驱动器已100％充满，因为对内核进行了许多更新。 默认情况下，ubuntu将保留旧内核并将它们添加到您可以在Grub2引导加载程序菜单中引导到的可用内核列表中

解决方案：

  1. 为了解决此问题并成功启动，请在启动过程中查看错误时（应该已经在控制台上），然后将vm或计算机重新启动到Grub2菜单中，然后选择“ubuntu的高级选项”视图 您可以在其中看到可以启动的旧内核的列表。
    可以在按住Shift键的情况下执行此引导，或者如果它是虚拟机，则应该能够在Grub启动屏幕中向下箭头并在启动时为ubuntu选择高级选项：

  2. 进入高级启动菜单后，您可能会看到列出的几个内核。 从最高/最高版本的内核中选择倒数第二个内核。 在我的情况下，我使用Linux 4.4.0-57-generic启动到标有Ubuntu的版本（下面的启动菜单屏幕截图很干净，但是您可能会看到列出的几个内核）。



以上的两个步骤是借鉴了这一篇博客。所以跟我的第一张图的不太一致，但是结果亲测有效。完成以上两个步骤以后，电脑就会自动重启。
